We are starting Phase 2.5 — Brand Studio. This is a new admin section where all brand knowledge, Gukdung's profile, and content guidelines are stored in our own database. This replaces storing prompts in Make.com.

## Step 1: Database Schema

Add the following tables to `shared/schema.ts`:

New table `brand_context`:
- id: uuid primary key default random
- type: text not null — "gukdung_profile" | "brand_voice" | "campaign_memory" | "post_guideline"
- title: text not null
- content: text not null
- isActive: boolean default true not null
- createdAt: timestamp defaultNow
- updatedAt: timestamp defaultNow

New table `gukdung_images`:
- id: uuid primary key default random
- imageUrl: text not null
- description: text nullable
- tags: text nullable — comma-separated e.g. "beach,birthday,indoor"
- isTrainingData: boolean default false not null
- loraModelId: text nullable
- createdAt: timestamp defaultNow

Add these columns to existing `marketing_queue` table:
- imageUrl: text nullable
- videoUrl: text nullable
- contentType: text nullable — "photo" | "reel" | "tiktok" | "card"
- scheduledAt: timestamp nullable
- postedAt: timestamp nullable
- postId: text nullable
- imagePrompt: text nullable

After schema changes, run: npm run db:push

---

## Step 2: Storage Methods

Add to `server/storage.ts`:

Brand Context CRUD:
- getBrandContextItems() — return all brand_context rows
- getBrandContextByType(type: string) — filter by type
- createBrandContextItem(data) — insert new row
- updateBrandContextItem(id, data) — update by id
- deleteBrandContextItem(id) — delete by id

Gukdung Images CRUD:
- getGukdungImages() — return all gukdung_images rows
- createGukdungImage(data) — insert new row
- updateGukdungImage(id, data) — update by id
- deleteGukdungImage(id) — delete by id

## Step 3: API Routes

Add to `server/adminRoutes.ts`, all protected by requireAdmin middleware:

GET    /api/admin/brand/context         — list all brand_context items
POST   /api/admin/brand/context         — create brand_context item
PATCH  /api/admin/brand/context/:id     — update brand_context item
DELETE /api/admin/brand/context/:id     — delete brand_context item

GET    /api/admin/brand/images          — list all gukdung_images
POST   /api/admin/brand/images          — create gukdung_image record
PATCH  /api/admin/brand/images/:id      — update gukdung_image record
DELETE /api/admin/brand/images/:id      — delete gukdung_image record
