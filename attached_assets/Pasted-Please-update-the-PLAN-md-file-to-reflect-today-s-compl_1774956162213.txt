## Add to Phase 2 (Marketing Command Center) — mark as COMPLETED

### Make.com Integration — Completed 2026-03-31

**Make.com Scenario (Integration Webhooks):**
- Flow: Custom Webhook → Anthropic Claude (Sonnet 4.6) → HTTP module → our server
- Claude module: Korean prompt instructing copywriting for Spoilt Dogs brand
  - Input variables: topic, prompt_text, platform (from webhook payload)
  - Output: Full social media post with caption + hashtags + emojis
- HTTP module settings:
  - URL: https://spoiltdogs.com.au/api/admin/marketing/queue
  - Method: POST
  - Header: x-api-key: {ADMIN_PASSWORD}
  - Body fields: caption (mapped to Claude Text Response), topic, platform, status: "pending"
- Data Structure "Spoiltdogs": topic (text), caption (text), platform (text), status (text)

**Issues resolved during integration:**
- 401 Unauthorized: required admin session login on production before testing
- generated_content field empty: Claude prompt was confused by JSON formatting instructions → reverted to simple Korean prompt
- "caption:" prefix appearing: removed hardcoded text from HTTP body field, left only the variable chip
- extractJsonFromClaudeOutput() helper function added to adminRoutes.ts for robust JSON parsing

**Progress bar added to Freeform Post dialog (admin-marketing.tsx):**
- Timer-based progress bar: 0% → 90% over 30 seconds
- Polling-based completion detection: checks queue every 2 seconds for new items
- On new item detected: jumps to 100%, shows success toast, closes dialog automatically
- Labels update during progress: "Make.com에 전송 중..." → "Claude가 콘텐츠를 작성하고 있습니다..." → "거의 다 됐어요..."
